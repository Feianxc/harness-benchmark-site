## Design Context

### Users
- 普通用户：正在使用 Claude Code / Codex / OpenCode，不想读协议细节，只想快速判断先试哪个 harness。
- 评测参与者：需要提交、校验、复验和追溯证据，关心是否能进入 Official / Frontier / Community 三层。
- 站点运营者：当前默认是单人、低预算运营，需要优先保护真实性边界，而不是依赖昂贵全量复跑。

### Brand Personality
- 克制、可信、清醒。
- 像 benchmark site / analyst terminal，而不是泛 SaaS 营销页。
- 先给结论，再给证据；对 demo、策展、verified truth 的边界保持诚实。

### Aesthetic Direction
- Light-first、editorial benchmark、低装饰、高信息密度。
- 参考方向：Artificial Analysis / Arena Leaderboard 的“数据先行”，但避免紫蓝渐变、玻璃卡片和模板化 hero metric。
- Consumer 层要像导购决策路由；Evidence 层要像审计台和协议浏览器。

### Design Principles
1. 默认答案优先：首屏先回答“我该先试哪个 harness”。
2. 证据边界可见：curated demo、runtime public、verified evidence 必须一眼分清。
3. 版式少卡片化：少用装饰卡片，更多使用表格、分栏、细线、密度和留白建立秩序。
4. 文案短而具体：避免“赋能、智能、下一代、强大”等泛化词；每句话说明范围、状态或下一步。
5. 移动端不缩水功能：移动端优先保持导航、榜单和证据入口可达。
