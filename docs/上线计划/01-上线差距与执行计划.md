# Harness 测评网站上线差距与执行计划（v0.1 → public beta）

生成时间：2026-04-24  
适用范围：`E:/工作区/10_Projects_项目/harness测评网站`  
当前定位：**Consumer selection guide + Evidence/Protocol public beta**。Consumer 榜单仍是 `curated_host_fit_demo`，不能宣称为 verifier-backed 真实主榜。

## 1. 上线结论

当前项目已经具备本地演示与 public beta 雏形，但距离正式上线还缺 5 类能力：

1. **发布外壳**：健康检查、安全响应头、robots/sitemap、部署说明、环境变量说明。
2. **数据治理**：默认示例数据、真实上传数据、worker snapshot、运营治理文件之间需要明确上线流程。
3. **前端质感**：当前信息架构已成型，但局部仍有泛 SaaS / AI 味，包括渐变、重卡片、过多说明文和重复 CTA。
4. **可信运营**：需要把“谁能上榜、谁被暂停、如何复核、如何纠错”写成 operator checklist。
5. **发布验证**：需要固定 build/test/smoke/a11y/移动端截图的上线门禁。

## 3. 本轮执行范围

本轮只做低风险、高收益、不会改变协议语义的上线准备：

1. 新增 `.impeccable.md` 设计上下文。
2. 新增本上线计划文档。
3. Web server 增加健康检查、安全头、robots/sitemap。
4. 前端 CSS 去掉明显 AI 味：重渐变、重阴影、卡片顶部色条、Inter 默认感。
5. Consumer 文案小幅收紧：更像 benchmark selection guide，少营销口吻。
6. 更新/补充测试，跑 build/test。

## 4. 本轮不做

- 不接入真实生产数据库。
- 不改 consumer demo 为真实 verified board。
- 不新增付费云服务。
- 不做复杂登录、账号、支付或管理后台。
- 不做全量复跑机制；仍按低预算抽样复核路线推进。

## 6. 剩余上线建议顺序

1. 先把当前单机 web public beta 部署起来，仅展示 consumer guide + evidence beta。
2. 再补 operator checklist、terms/privacy、submission policy。
3. 然后添加 CI/CD 与部署脚本。
4. 外部上传只开放候选池；receipt 不进入榜单，Evidence/Official 仍走 intake gate 和 governance state machine。

## 7. 本轮执行结果（2026-04-24）
