# 开源仓库与同步方案

生成时间：2026-04-24

## 1. 本地 Git 边界

本项目将初始化为 Git 仓库，默认分支：`main`。

必须纳入版本管理：

- `apps/`
- `packages/`
- `registry/`
- `docs/`
- `.codex/` 项目 agent 配置
- `apps/verifier-worker/data/` 公开演示快照（只允许脱敏后的 public beta snapshot）
- `README.md`
- `LICENSE`
- `SECURITY.md`
- `CONTRIBUTING.md`
- `Dockerfile`
- `.env.example`
- `.github/workflows/ci.yml`

必须排除：

- `node_modules/`
- `.data/`
- `.workspace/`
- `.hb/`
- `apps/mock-intake/data/`
- `.env` / `.env.*`
- sealed traces / private payloads / runtime receipts

特别边界：

- `.codex/config.toml` 只保留项目可复现配置，不放本机 `C:/Users/...` 绝对路径。
- `apps/verifier-worker/data/` 当前作为公开 demo snapshot 入库；如果未来改成生产 worker 运行目录，应改用 `OHBP_WORKER_DATA_DIR` 指向持久化目录，并把生产数据目录排除出 Git。

## 2. GitHub 官方流程

GitHub 官方文档给出的本地项目导入流程是：初始化 Git 仓库、添加文件、提交、添加远程仓库、push。

参考：

- https://docs.github.com/en/migrations/importing-source-code/using-the-command-line-to-import-source-code/adding-locally-hosted-code-to-github?platform=windows
- https://docs.github.com/articles/adding-a-remote?platform=windows

## 3. 本地命令

```bash
git init -b main
git add .
git commit -m "Initial public beta release"
```

## 4. 远程仓库

当前已创建并推送公开仓库：

```text
https://github.com/Feianxc/harness-benchmark-site
```

当前默认分支：

```text
main
```

如果你已经有 GitHub 空仓库：

```bash
git remote add origin https://github.com/<OWNER>/<REPO>.git
git push -u origin main
```

如果使用 GitHub CLI 且已登录：

```bash
gh repo create <OWNER>/<REPO> --public --source=. --remote=origin --push
```

## 5. 自动同步建议

仓库已放入最小 CI 文件：

```text
.github/workflows/ci.yml
```

内容等价于：

```yaml
name: verify
on:
  push:
    branches: [main]
  pull_request:

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: npm
      - run: npm ci
      - run: npm run build
      - run: npm run test
      - run: npm run lint
```

## 6. 发布前敏感信息检查

```bash

git check-ignore -v .data .workspace .hb .env
git status --ignored --short
```
